# Amazon S3 (Simple Storage Service)

## 1. Tổng quan (Overview)

Amazon S3 là dịch vụ lưu trữ đối tượng (object storage) cung cấp khả năng mở rộng, khả năng sẵn sàng dữ liệu, bảo mật và hiệu năng hàng đầu.

- **Object Storage**: Lưu trữ dữ liệu dưới dạng objects (file) trong buckets (thùng chứa).
- **Durability**: 99.999999999% (11 số 9). Xác suất mất file cực thấp.
- **Availability**: 99.99% (S3 Standard).

## 2. Các lớp lưu trữ (Storage Classes)

1.  **S3 Standard**:
    - Truy cập thường xuyên (Frequently Accessed).
    - Độ trễ thấp (miliseconds), throughput cao.
    - _Use Case_: Host website tĩnh, Big Data, Content Distribution.

2.  **S3 Intelligent-Tiering**:
    - Tự động di chuyển dữ liệu giữa các lớp truy cập (Frequent/Infrequent) dựa trên patterns sử dụng thực tế.
    - Phí monitoring nhỏ, nhưng không có phí retrieval.
    - _Use Case_: Dữ liệu không rõ pattern truy cập.

3.  **S3 Standard-IA (Infrequent Access)**:
    - Dữ liệu ít truy cập, nhưng cần lấy nhanh khi cần.
    - Giá rẻ hơn Standard, nhưng tốn phí retrieval ($/GB).
    - _Use Case_: Backup, Disaster Recovery.

4.  **S3 One Zone-IA**:
    - Lưu trữ trong **1 Availability Zone** (các loại khác là >= 3 AZs).
    - Rủi ro mất dữ liệu nếu AZ sập. Rẻ hơn Standard-IA 20%.
    - _Use Case_: Backup dữ liệu không quan trọng (có thể tạo lại được).

5.  **S3 Glacier (Archive)**:
    - _Instant Retrieval_: Lấy ngay lập tức (ms). Đắt nhất trong Glacier.
    - _Flexible Retrieval_: Lấy trong 1-5 phút hoặc 3-5 giờ.
    - _Deep Archive_: Lấy trong 12-48 giờ. Rẻ nhất.

## 3. Các tính năng chính (Key Features)

### a. Versioning

- Lưu giữ nhiều phiên bản của một object.
- Bảo vệ khỏi xóa nhầm hoặc ghi đè (Overwrite).
- Có thể khôi phục file đã xóa (Delete Marker).

### b. Replication

- **CRR (Cross-Region Replication)**: Sao chép sang Region khác (Compliance, DR, Low latency access).
- **SRR (Same-Region Replication)**: Sao chép sang Bucket khác cùng Region (Log aggregation, Test env).
- Yêu cầu: Phải bật Versioning ở cả Source và Destination.

### c. Lifecycle Policies

- Tự động chuyển object sang lớp lưu trữ rẻ hơn (Standard -> IA -> Glacier) hoặc xóa hẳn sau X ngày.

### d. Security

- **Server-Side Encryption (SSE)**: S3 tự động mã hóa.
  - _SSE-S3_: AWS quản lý key.
  - _SSE-KMS_: KMS quản lý key (Audit được ai dùng key).
  - _SSE-C_: Khách hàng quản lý key.
- **Bucket Policies**: JSON policy kiểm soát quyền truy cập (cho phép public hoặc chỉ cho VPC cụ thể).
- **Block Public Access**: Chặn hoàn toàn việc public bucket (nên bật mặc định).

## 4. Exam Tips (Lưu ý thi SAA)

- Nếu đề bài yêu cầu lưu trữ **giá rẻ nhất** cho dữ liệu **lâu dài (archive)** -> Chọn **S3 Glacier Deep Archive**.
- Nếu muốn **website tĩnh (serverless)** -> Chọn **S3 Static Website Hosting**.
- Nếu muốn **tối ưu chi phí** cho dữ liệu không rõ tần suất truy cập -> Chọn **S3 Intelligent-Tiering**.
- Để **chống xóa nhầm (accidental deletion)** -> Bật **Versioning** và **MFA Delete**.
- Để **khóa file** không cho sửa/xóa trong 1 năm (WORM) -> Dùng **S3 Object Lock** (Governance/Compliance mode).
- Để tăng tốc độ upload file lớn từ khắp thế giới -> Dùng **S3 Transfer Acceleration**.
